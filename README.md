## Development Policy

In the database design of this repository, an immutable data model is adopted as a general rule.

## Database Schema Changes

`Database/schema.sql` is the desired table/index/constraint schema.
`Database/extensions.sql` contains PostgreSQL extensions needed before loading
that schema. `Database/migrations` contains the versioned SQL migrations that
move existing databases toward that schema.

To change the schema:

```sh
# 1. Edit Database/schema.sql
# 2. Prepare an Atlas dev database and install required extensions
psql "$ATLAS_DEV_DATABASE_URL" -f Database/extensions.sql

# 3. Generate a migration with Atlas
atlas migrate diff schema --env local --var "dev_url=$ATLAS_DEV_DATABASE_URL"

# 4. Review the generated SQL under Database/migrations and run tests
swift test
```

Migrations are generated by Atlas and checked by the Atlas GitHub Actions
workflow. On deployment, the deploy workflow applies committed SQL migrations
with Atlas before updating the Cloud Run service image. The application does
not apply SQL migrations at startup; database initialization is owned by Atlas
and the local Postgres init scripts. CI and deploy install the Atlas Community
Edition.

This project uses forward-only migrations. Down migrations are not generated.
If a migration must be corrected, add a new forward migration.

The Atlas workflow starts a PostgreSQL service in GitHub Actions, installs
required extensions, and passes its URL to Atlas via `dev_url`. Production
migration apply needs `POSTGRES_HOSTNAME`, `POSTGRES_USER`, and `POSTGRES_DB`
GitHub repository variables, and reads `POSTGRES_PASSWORD` from Google Secret
Manager.

When adopting Atlas for an existing production database that already contains
the baseline schema, set `ATLAS_BASELINE_VERSION=000001` for the first deploy.
Atlas then records `000001_initial_schema.sql` as already applied and runs only
future migrations. For a new empty database, leave `ATLAS_BASELINE_VERSION`
unset so Atlas applies the baseline SQL normally.

## Run on Local

- Postgres

```sh
container run --rm \
  --name blindlog-postgres \
  -e "POSTGRES_PASSWORD=test_password" \
  -e "POSTGRES_USER=test_user" \
  -e "POSTGRES_DB=test_database" \
  -p 5432:5432 \
  postgres:17.6
```

- Valkey

```sh
container run --rm \
  --name blindlog-valkey \
  -p 6379:6379 \
  valkey/valkey:8.1.4
```

- OpenTelemetry Collector

```sh
container run --rm \
  --name blindlog-otel-collector \
  -p 4317:4317 \
  -p 4318:4318 \
  -v "$PWD/Deploy/local/otel-collector-config.yaml:/etc/otelcol-contrib/config.yaml" \
  ghcr.io/open-telemetry/opentelemetry-collector-releases/opentelemetry-collector-contrib:0.144.0
```
